임원용 경쟁사 리서치 대시보드를 React 앱으로 완성하는 실습

이번 워크숍은 Gems에서 Build용 초안 프롬프트를 만들고, AI Studio Build에서 리서치 기능을 붙인 대시보드를 생성한 뒤, refinement, LG 스타일과 웹 스타일 요청, 외부 API 확장, 공유까지 이어가는 흐름을 다룬다.

다운로드
이 실습의 핵심
• 시작점은 앱 구현이 아니라 Gems에서 실습용 프롬프트 초안을 만드는 것이다.
• Build에서는 완성보다 같은 대화에서 계속 refinement하는 감각을 익힌다.
• 기능 요청, 스타일 요청, API 확장을 분리하고 마지막에는 XML 구조 프롬프트로 요구사항을 정리한다.
• 결과 앱을 동료에게 공유하거나 이후 API 연동 확장으로 이어갈 수 있는 기반을 만든다.

Gems + Build 5단계 실습 흐름

  

    

      

        
Step 0

        Gems로 초안 프롬프트 설계
        
비즈니스 목표와 사용자, 기능을 정리해 Build에 넣을 실습용 초안 프롬프트를 먼저 만든다. 시작점은 코드가 아니라 좋은 brief다. 필요하면 Gems를 짧은 프롬프트 정리 보조 앱처럼 써서 brief를 XML이나 Build용 요청문으로 다시 써도 된다.

      

      

        
Step 1

        리서치 대시보드 초안 생성
        
경쟁사 뉴스와 기술 동향을 검색·요약하는 첫 React 대시보드를 만든다. 중요한 것은 “동작하는 초안”을 빠르게 얻는 것이다.

      

      

        
Step 2

        리서치 기능 refinement
        
KPI 카드, 중요도 태그, 정렬, 북마크 같은 기능을 같은 대화 안에서 차례로 추가한다. 여기서 Vibe Coding의 진짜 힘이 보인다.

      

      

        
Step 3

        LG 스타일과 웹 스타일 적용
        
LG 매젠타 포인트, 절제된 카드 UI를 적용하고, 필요하면 스타일 가이드에서 고른 웹 스타일 요청을 얹어 화면 성격을 정리한다.

      

      

        
Step 4

        선택 확장: 외부 API와 공유
        
Firecrawl 같은 API를 연결해 더 넓은 소스를 끌어오도록 확장하고, 마지막에는 Share 또는 Publish로 결과를 공유한다.

      

    

    

      
스타일 요청도 실습에 포함한다

      
같은 기능이라도 “Pure Minimal Grid로”, “Scandinavian System으로” 같은 요청을 더하면 화면 결과가 크게 달라진다. 스타일 선택지는 웹 스타일 프롬프트 가이드에서 바로 참고할 수 있다.

구조화된 프롬프트를 한 번 더 써보자

  

    
처음부터 XML을 손으로 길게 쓰기보다, Gems에서 비즈니스 brief를 먼저 정리하고 그 결과를 XML로 다시 묶는 흐름이 실습에서는 가장 안정적이다. 아래 예시는 전체 파일의 핵심만 추린 것이다.

    

      
XML 프롬프트 예시

      
<app_request>
  <goal>Create an executive competitive-intelligence dashboard</goal>
  <framework>React</framework>
  <data_flow>Search recent news, summarize findings, assign urgency, and prepare export-friendly cards.</data_flow>
  <brand_style>LG magenta accent, restrained executive tone, white background.</brand_style>
</app_request>

    

    

      
언제 XML을 넣는 것이 가장 좋은가

      
초안 프롬프트를 만든 직후보다, 한두 번 refinement를 거쳐 요구사항이 굳은 다음 XML로 정리하는 편이 효율적이다. 필요하면 Gems에 “이 프롬프트를 app_request XML 형식으로 다시 써줘”라고 맡겨 가벼운 XML 변환 보조 앱처럼 쓰는 정도면 충분하다.

완료 기준

      

        1. 검색 입력과 결과 카드가 동작한다.

        2. KPI 또는 요약 카드가 추가되어 있다.

        3. LG 스타일 지시문이 반영되어 화면 위계가 정리되어 있다.

        4. 필요하면 Share 또는 Publish로 다른 사람에게 보여줄 준비가 되어 있다.
