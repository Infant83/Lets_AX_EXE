오늘은 React를 사용한다
Advanced settings에서 프레임워크를 고를 수 있지만, 오늘 실습은 React를 기준으로 진행한다. 컴포넌트 단위로 고치기 쉽고, AI와의 반복 대화에서 수정 방향이 가장 직관적이기 때문이다.
실제 화면으로 보는 Build 인터페이스
Advanced settings에서는 모델, 시스템 지침, 프레임워크, 마이크 입력 등을 정한다. 여기서 React / Next.js / Angular 중 하나를 고를 수 있다.
실제 생성된 앱 화면이다. 프롬프트를 주면 오른쪽 preview가 바뀌고, 같은 대화에서 계속 기능을 덧붙이며 앱을 발전시킬 수 있다.
Advanced settings에서 볼 5가지
항목무엇을 정하나오늘 실습 기준
Model어떤 Gemini 엔진으로 앱 생성과 대화를 할지기본 추천 모델을 쓰되, 응답 품질이 아쉬우면 상위 모델로 바꾸는 정도만 익힌다.
System instructions앱의 역할과 성격, 작성 규칙“임원용 보고 앱”, “LG 스타일”, “React로 작성” 같은 기준을 넣는다.
Framework어떤 웹 프레임워크로 앱을 생성할지React를 선택한다. 반복 수정과 코드 확인이 가장 직관적이다.
Preview / Assistant채팅과 결과 화면을 어떻게 같이 볼지한쪽에서 수정 요청, 다른 쪽에서 바로 결과 검수하는 리듬을 만든다.
Share / Publish앱을 동료와 공유할지, 공개 링크로 배포할지내부 실습은 공유 링크 중심, 외부 공개는 Publish 패널을 따로 본다.
React / Next.js / Angular를 어떻게 구분하면 되나
React는 빠른 프로토타이핑과 컴포넌트 수정에 유리해서 오늘 실습용으로 가장 적합하다.
Next.js는 라우팅, 서버 기능, 정식 배포까지 염두에 둘 때 좋다.
Angular는 기업 표준과 복잡한 대형 화면 구조를 강하게 통제해야 할 때 어울린다.

공유와 Publish는 다르다
Share는 특정 동료와 앱을 같이 보는 협업 설정이다. 링크 공유나 사람별 권한 지정이 이 단계다.
Publish는 공개 URL 배포에 가깝다. Notebook 기록이나 코드 편집 내역은 private로 두고, 결과 앱만 외부에 보여주는 흐름으로 이해하면 된다.
