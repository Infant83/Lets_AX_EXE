Google AI Studio API 키 발급

오늘 수업에서 API 키를 깊게 쓰지는 않지만, 이후 AI Studio 앱을 내려받아 실행하거나 API 기반 실습으로 넘어갈 때를 위해 발급 위치와 보관 원칙만 간단하게 정리한다.
현재 UI 기준 발급 위치
현재 AI Studio에서는 Dashboard > API 키 화면 상단에서 API 키 만들기로 진입한다. 키 목록 전체는 개인 정보가 섞일 수 있으므로 수업 자료에는 상단 안내 영역만 넣었다.
Quick Path
  1. aistudio.google.com에 로그인한다.
  2. 좌측 하단 또는 상단 메뉴의 Get API key로 이동한다.
  3. API 키 만들기를 눌러 연결할 프로젝트를 선택한다.
  4. 발급된 키는 복사만 하고, 코드에는 직접 쓰지 말고 환경 변수로 넣는다.
발급 후 바로 지켜야 할 규칙
1. 코드에 직접 넣지 않기
키를 소스 코드나 슬라이드에 그대로 적으면 유출 위험이 커진다. 문서 공유 전에는 항상 제거한다.
2. 환경 변수로 분리
보통은 GEMINI_API_KEY처럼 운영체제 환경 변수나 `.env` 파일로 연결해 앱과 비밀값을 분리한다.
3. 프로젝트와 함께 관리
같은 키라도 어느 프로젝트에 연결했는지에 따라 quota, billing, 접근 가능한 기능이 달라질 수 있다.
4. 유출 시 즉시 폐기
메신저나 공개 저장소에 노출됐다면 기존 키는 폐기하고 새 키를 발급받는 것이 원칙이다.
예시 연결 방식
로컬에서 앱을 실행할 때는 보통 아래처럼 환경 변수로 넣는다. 
GEMINI_API_KEY=your_api_key_here
오늘 수업에서는 여기까지만

오늘은 API 키를 여러 번 쓰는 실습을 진행하지 않는다. 대신 아래 두 가지만 기억하면 충분하다.

  • AI Studio Build 앱을 나중에 로컬이나 사내 환경에서 실행하려면 결국 API key 연결이 필요하다.
  • 키는 기술 설정이면서 동시에 비용, 보안, 운영 정책의 출발점이다.
