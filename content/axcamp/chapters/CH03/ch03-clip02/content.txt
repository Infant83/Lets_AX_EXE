프롬프팅 기초
AI에게 효과적으로 지시하는 핵심 프레임워크와 프롬프트 작성 기법을 배웁니다.
AI에게 효과적으로 지시하는 7가지 요소 (Delegation Canvas)

좋은 프롬프트의 본질은 "AI에게 업무를 위임하는 지시서"를 잘 작성하는 것입니다.

#	요소	프롬프트 예시	CEO 예시
1	Role (역할)	"당신은 어떤 역할인가?"	전략 참모 / 재무 분석가 / 보고서 작성자
2	Goal (목표)	"무엇을 해달라는 것인가?"	분석 / 비교 / 요약 / 작성 / 번역
3	Context (맥락)	"배경 상황은 무엇인가?"	LG그룹 계열사 사업 맥락 / 이사회 안건 / 경쟁사 대응
4	Constraints (제약)	"반드시 지켜야 할 조건은?"	기밀 제외 / 2페이지 이내 / 격식체
5	Output (형식)	"어떤 형태로 제시할 것인가?"	표 + 매트릭스 / 1페이지 요약 / 옵션 3개
6	Evaluation (평가)	"좋은 결과의 기준은 무엇인가?"	ROI 포함 / 리스크 분석 / 경쟁사 비교
7	Questions (선제 질문)	"더 나은 결과를 위해 먼저 물어볼 것"	예산 범위? 우선순위? 타겟 시장?
CEO 실전 팁
7가지를 모두 포함할 필요는 없습니다. 처음엔 Role + Goal + Output 3개만 넣어도 답변 품질이 눈에 띄게 향상됩니다. 익숙해지면 Context와 Constraints를 추가하세요.
프롬프트 엔지니어링 4원칙

Delegation Canvas를 실전에 적용할 때, 특히 중요한 4가지 원칙입니다.

원칙	설명	CEO 활용 예시
Format (형식)	출력 형태를 구체적으로 지정하면 AI가 즉시 활용 가능한 결과를 생성합니다	"표 형태로", "A4 1장 분량으로", "불릿 포인트로"
Role (역할)	AI에게 전문가 역할을 부여하면 관점과 깊이가 달라집니다	"전략 참모로서", "재무 분석가로서", "비서관으로서"
Context (맥락)	파일 첨부로 풍부한 배경 정보를 제공하면 정확도가 크게 향상됩니다	회의록 첨부, 보고서 첨부, 전략 문서 첨부
Step (단계)	하나의 긴 프롬프트보다, 대화를 이어가며 단계적으로 깊이를 더합니다	요약 → 이슈 추출 → 전략 제안 → 시나리오 분석
4원칙 적용 순서
Format → Role → Context → Step 순서로 프롬프트를 구성하면 자연스럽게 좋은 결과를 얻을 수 있습니다. 처음엔 Format과 Role만 적용해도 답변 품질이 크게 달라집니다.
컨텍스트 엔지니어링 연결
프롬프트 엔지니어링 4원칙은 컨텍스트 엔지니어링의 실천 도구입니다. 역할·맥락·형식·단계적 사고를 통해 AI에게 더 풍부한 컨텍스트를 제공하는 것이 곧 더 좋은 결과로 이어집니다.
